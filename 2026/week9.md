当前任务
基于 PicoZense DCAM710 深度相机的移动机器人自主避障与环境建图系统开发
本周工作
1. 完成 Autolabor Pro1 小车 1 小时循环运动脚本开发，实现 "后退 3 米→旋转 360 度→前进 3 米" 的周期性运动，速度、距离、运行时长等参数可一键配置
2. 解决深度相机硬件独占核心问题：排查发现原厂 FrameViewer 调试工具会 100% 独占相机 USB 硬件，导致 ROS 驱动无法访问，实现了程序退出时自动杀死残留进程、释放硬件的机制
3. 解决 ROS 环境兼容性问题：定位到 conda base 环境与 Ubuntu 18.04 自带的 ROS Melodic Python2.7 存在严重冲突，关闭 conda 自动激活后解决了 rospy、rospkg 等核心包找不到的问题
4. 开发专属 C++ 版 ROS 驱动：由于原厂 SDK 未提供 Python 接口，基于原厂 C++ 底层库开发了 PicoZense 专属 ROS 驱动节点，实现了深度图 (16UC1)、RGB 图和相机内参的标准 ROS 话题发布，帧率稳定在 30fps
5. 完成避障逻辑与 ROS 驱动的联调：实现前进过程中的实时障碍物检测，当障碍物距离小于 40cm 安全阈值时自动紧急停车，后退 20cm 避让，障碍物离开后自动恢复原运动流程
下周计划
1. 完善 PicoZense ROS 驱动，增加深度范围切换、背景滤波阈值等参数的动态配置功能
2. 部署 RTAB-Map SLAM 算法，完成与深度相机 ROS 话题的适配，实现基于单目 + 深度的实时 2D 栅格地图和 3D 点云地图构建
3. 完成自主避障系统与 SLAM 建图系统的联调，实现小车一边自动循环运动一边实时构建环境地图
4. 在实验室不同场景下测试建图精度和避障可靠性，优化相机内参、避障阈值等关键参数
