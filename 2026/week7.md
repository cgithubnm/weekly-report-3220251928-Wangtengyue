- 解决 ToDesk 远程键盘输入被识别为普通字符的核心问题，定位到 Windows PowerShell 行缓冲模式与 ROS 键盘控制程序的兼容性冲突

- 部署 ROS 官方 rqt\_robot\_steering 图形化控制工具，实现纯鼠标远程控制，适配 ToDesk 远程操作场景

- 完成底盘驱动与 ROS 环境的兼容性调试，确保远程控制指令能稳定下发至小车底盘

## 二、问题与解决方案

|问题描述|根因分析|解决方案|
|---|---|---|
|ToDesk 远程键盘输入被识别为字符|PowerShell 行缓冲模式，按键需按回车才发送|改用 Windows Terminal 的 SSH 连接 \+ ROS rqt 图形控制|
|Minimax API 返回 unknown model 错误|v2 接口不兼容 v1 的 abab6 模型名|替换为 Trae 平台验证过的 MiniMax\-M2\.7 模型|
|自然语言控制鲁棒性差，部分指令不执行|提示词约束不足，模型输出格式不稳定|重写提示词，增加强制 JSON 格式与数值规则|
|大模型响应延迟高，无法满足实时控制|纯文本大模型推理 \+ 网络传输延迟达 1\-3 秒|采用分层架构，底层用 ROS 算法处理实时控制|
